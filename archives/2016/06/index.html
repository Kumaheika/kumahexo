使用 Hexo 來寫 Blog(遇到的問題)

node.js Hexo sass?

本篇主要紀錄載上傳時遇到的問題

上一篇主要是介紹上傳到github過程，本篇是說明一些遇到的問題

上傳 github 後網頁沒吃到CSS樣式？

剛上傳時候，很開心地跑上去連結一下成果，結果會發現網頁竟然沒吃到CSS樣式，
詢問過同事之後，才知道上一篇_config.yml 裡面設定，如果 : 後面不加上空格的話，會導致連結失效以致於吃不到 CSS (如下)

#URL 設定 ( : 後面要有空格 )
url: https://github.com/你的github名稱
root: /repository的名稱/
#Deployment 設定 ( : 後面要有空格 )
deploy:
    type: github
    repository: https://github.com/你的github名稱/kumahexo.git
    branch: gh-pages (如果是用github主頁就不用)

更新了 theme ，上傳還是舊的？

這問題一開始也困擾我很久，後來爬了一下發文才發現到，
因為 HEXO 會自動判斷你的檔案是否有新增減少，
如果只是改 theme 在 generate 時候只會更新_posts裡面的內容，
為了讓他每次上傳都記得更新，所以在 generate 之前要先執行

這樣才能確保每次上傳都是最新的狀態。

使用 Hexo 來寫 Blog(上傳github)

node.js Hexo sass?

本篇主要紀錄載上傳github的設定

上一篇主要是介紹安裝的過程，本篇主要是紀錄上傳到github過程

上傳 github 的設定

在上傳前，請先開啟根目錄下 _config.yml 做一些基本的設定

#URL 設定 ( : 後面要有空格 )
url: https://github.com/你的github名稱
root: /repository的名稱/
#Deployment 設定 ( : 後面要有空格 )
deploy:
    type: github
    repository: https://github.com/你的github名稱/kumahexo.git
    branch: gh-pages (如果是用github主頁就不用)

完成以後，在 commandLine (mac是終端機)，移動到目前 HEXO 的資料夾下，

輸入

HEXO 就會開始將資料 build 成可上傳的檔案，這時 HEXO 內就會多出 public 的文件夾 ，最後在執行上傳的動作

就會開始上傳至 github 了！！
